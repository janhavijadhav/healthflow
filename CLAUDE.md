# HealthFlow ‚Äî Claude Code Reference

## What This Project Is

HealthFlow is a healthcare data engineering pipeline on AWS. It ingests synthetic patient data (generated by [Synthea](https://synthea.mitre.org/)) through a 3-zone S3 data lake (raw ‚Üí processed ‚Üí curated), with downstream layers for transformation, orchestration, serving, and monitoring that are scaffolded but not yet implemented.

**Environment**: `dev` on AWS account `805046891516`, region `us-east-1`.

## AWS Architecture

### Data Lake (S3, 3-zone)
| Zone | Bucket | Purpose |
|------|--------|---------|
| Raw | `healthflow-raw-dev-805046891516` | Landing zone ‚Äî partitioned by `table/year=/month=/day=` |
| Processed | `healthflow-processed-dev-805046891516` | Cleaned/typed data |
| Curated | `healthflow-curated-dev-805046891516` | Analytics-ready aggregates |

### Ingestion Flow
1. `upload_to_s3.py` reads local Synthea CSVs from `data/sample/` and uploads them to the raw bucket with date partitioning and a manifest JSON under `_manifests/`.
2. Any `PutObject` on the raw bucket under the `claims/` prefix fires the Lambda (`healthflow-ingestion-trigger-dev`).
3. Lambda logs S3 event metadata to CloudWatch (`/aws/lambda/healthflow-ingestion-trigger-dev`, 7-day retention). A CloudWatch alarm fires when Lambda errors > 0.
4. `validate_raw.py` can be run locally to pull today's partition from S3 and run basic checks (has rows, no empty column names) on: `patients`, `encounters`, `conditions`, `medications`.

### Glue
- Catalog DB: `healthflow_dev`
- `healthflow-raw-crawler-dev`: crawls `raw/claims/` and `raw/patients/` on a daily 6 AM UTC schedule
- `healthflow-processed-crawler-dev`: crawls `processed/claims/` (no schedule yet)

## Data Tables (Synthea)

The 18 source tables in `data/sample/`:
`patients`, `encounters`, `conditions`, `medications`, `procedures`, `observations`, `allergies`, `careplans`, `claims`, `claims_transactions`, `devices`, `imaging_studies`, `immunizations`, `organizations`, `payer_transitions`, `payers`, `providers`, `supplies`

The active ingestion targets 6 of these: `patients`, `encounters`, `conditions`, `medications`, `procedures`, `observations`.

## Key Conventions

- Resource naming: `{project_name}-{resource}-{environment}` (e.g. `healthflow-raw-crawler-dev`)
- S3 partitioning: `{table}/year={YYYY}/month={MM}/day={DD}/`
- All Terraform tags use `Project` and `Environment` keys
- Lambda runtime: Python 3.12
- Log format: structured JSON in Lambda, `%(asctime)s %(levelname)s %(message)s` in scripts
